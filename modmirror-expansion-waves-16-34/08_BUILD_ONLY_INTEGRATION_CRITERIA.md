# Build-Only Integration Criteria

At the end, create:

`docs/expansion-waves/BUILD_REPORT.md`

Include:
- waves implemented,
- waves deferred and why,
- files changed,
- branches/worktrees used,
- tests run,
- runtime checks run,
- safety gates,
- known gaps,
- recommended next engineering waves.

Do not include Devpost/submission copy.

Successful continuation should produce:
- no smoke-test production labels,
- post/comment operational entrypoints,
- action execution receipts or truthful fallback,
- richer scan history,
- drift trend data,
- policy ratification workflow,
- stronger Case Packets,
- optional verified AI advisory layer,
- truthful runtime capability matrix,
- meaningful tests and synthetic evaluation.
