# Wave 31: Mobile and Runtime Resilience

## Objective

Make essential flows usable in narrow/mobile WebViews and degraded runtime conditions.

## Deliverables

- responsive UI
- degraded-state UX
- error taxonomy
- tests/static checks

## Acceptance Criteria

- no critical horizontal overflow
- core actions reachable narrow
- errors actionable
- fallbacks labeled
