# Operational UI IA Spec

The UI must serve the real product workflow.

## Navigation

- Act
- Scan
- Agree
- Review
- Prove
- Settings

## Layout principles

- Use evidence/document/ledger/timeline layouts.
- Avoid generic card-grid AI slop.
- Keep visual hierarchy clear.
- Show verified vs inferred evidence.
- Make demo/live state obvious.
- Make execution mode obvious.

## Key surfaces

### Act
Target-aware Apply Policy. Shows target, policy, recommendation, evidence, action preview, confirmation, receipt.

### Scan
Live/demo scan controls, scan depth, scan history, drift candidates, data quality.

### Agree
Policy proposals, reviews, adoption state, version history.

### Review
Overrides, receipts needing review, policy health.

### Prove
Case Packets, digest, before/after consistency.

### Settings
Capability matrix, runtime proof, safety flags, data retention.
