# Hackathon Submission

Use this skill for public-facing ModMirror materials.

## Narrative

Lead with:

- consistency-first moderation
- enforcement drift discovery
- team policy agreement
- human-in-the-loop action guidance
- override audit as learning loop

Avoid generic "moderate faster" framing unless it supports consistency.

## Safe Claims

Use:

- "helps reveal enforcement drift"
- "helps teams define rule policies"
- "provides consistency nudges"
- "human-in-the-loop"
- "deterministic attribution with confidence scoring"

Avoid:

- "first ever mod notes tool"
- "first ever removal reason tool"
- "fully solves fairness"
- "automatically detects all rules correctly"
- "replaces moderators"

## Demo Assets

Submission should show:

- demo mode label
- Mirror Scan with Rule 2 drift
- policy ladder creation
- Apply Policy recommendation
- stricter/looser override nudge
- override audit record

Keep screenshots grounded in the actual implemented app. Do not write claims unsupported by working behavior.

## Final QA

Before submission, verify:

- `README.md` explains setup and limitations.
- `RESEARCH.md` documents Devvit limitations.
- `TODO.md` does not hide known blockers.
- screenshots/video use demo data or safe test subreddit data.
- build/check commands pass or failures are disclosed.
